MEMORY ARCHITECTURE — CHROMADB VECTOR STORE
Memory Accumulation Over Time
INT 1 User asks about Python →  Saved as vector to ChromaDB
INT 2 User asks about loops → ChromaDB returns Python context →  Answer references Python + loops
INT 3 User asks "give me a quiz" → ChromaDB returns Python + loops →  Quiz auto-targets Python loops!
Why Vector DB vs Traditional DB
TRADITIONAL DB
Exact keyword match only. Query "loops" finds only docs containing "loops". No semantic understanding.
CHROMADB VECTORS
Semantic similarity search. "Loops" finds iteration, for/while, repetition. Ranked by meaning proximity.

SESSION ID FLOW
Generated once on first visit as user_ + timestamp. Stored in localStorage. Sent as userId on every request. Ensures isolated ChromaDB collection per browser session.
