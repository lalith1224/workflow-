Project Overview

⊕ Project Objective

An AI-powered learning assistant that remembers you — powered by RAG, LangChain, and ChromaDB vector storage. Every interaction enriches the context for the next.

Summarize YouTube videos, refine notes, generate quizzes, and hold follow-up conversations with full episodic memory.

■ Constraints & Design Rules

  • ①Every request passes through the full RAG pipeline — no shortcuts.
  • ②No two sessions share the same ChromaDB collection — strict user isolation.
  • ③Context windows are capped to prevent overflow — 3 000 chars video, 2 000 chars quiz.
  • ④Prompt + response always saved after inference — memory grows with every turn.
Core RAG Pipeline — Every Request
→ 🖥️

Step 01

INPUT

User submits prompt via chat or feature button. Frontend sends POST /ask with taskType, prompt, userId, apiKey.

→ 🔍

Step 02

RETRIEVAL

Node server calls ChromaDB /memory/get. Cosine similarity returns top 3 semantically relevant past memories.

→ 🧩

Step 03

AUGMENT

LangChain assembles: System Persona + Retrieved Memories + Current Prompt → ChatPromptTemplate.

→ 🤖

Step 04

INFERENCE

Augmented prompt sent to Mistral Small 3.1 24B via OpenRouter API. Model generates a contextual response.

💾

Step 05

MEMORY

Prompt + Response stored as a new vector embedding in user's ChromaDB collection. Enriches future retrieval.

Feature Workflows
1

▶ Summarize Video

1

FRONTEND — User clicks button → analyzeVideo() called. Hooks into YouTube IFrame API to pull captions.

2

FALLBACK — No captions? Falls back to Watch History URLs stored in LocalStorage.

3

TRUNCATION — Caption text capped at 3 000 chars to prevent context-window overflow.

4

MEMORY — ChromaDB injects past relevant memories. Augmented prompt sent to Mistral.

5

RESPONSE — AI returns structured bullet summary, saved to ChromaDB for future context.

2

💬 Follow-up Chat

1

INPUT — User types vague follow-up: "explain the second point more"

2

VECTOR SEARCH — ChromaDB vectorizes phrase, finds cosine-similar past interactions.

3

INJECT — Previous summary returned as context. Mistral receives vague prompt + full prior context.

4

INFERENCE — Model infers "second point" = Question 2 of the previous response. Seamless continuation.

5

SAVE — This Q&A pair saved → richer context over time, building episodic memory.

3

📋 Refine Notes

1

COLLECTION — Scrapes cleanViewerNotes() from LocalStorage. Joins with \n formatting.

2

VALIDATE — If array length = 0, halts and alerts user. No empty API calls made.

3

PROMPT — AI instructed to group concepts, fix errors, structure beautifully, add 2 study tips.

4

OUTPUT — Mistral returns polished study guide with appended further-study recommendations.

4

🚗 Auto Quiz

1

SOURCE — Pulls latest notes from LocalStorage. Truncated to 2 000 chars for focused reasoning.

2

RAG BOOST — ChromaDB auto-injects video summaries. Quiz integrates both notes + watched content.

3

CONSTRAINTS — Model must: A/B/C/D format, only use provided context, obscure answers until bottom.

4

OUTPUT — 3 MCQ questions generated. Saved to memory for follow-up "explain Q2" continuity.

Memory Architecture — ChromaDB Vector Store

Memory Accumulation Over Time

INT
1

User asks about Python → saved as vector to ChromaDB

INT
2

User asks about loops → ChromaDB returns Python context → answer references both

INT
3

User asks "give me a quiz" → ChromaDB returns Python + loops → quiz auto-targets Python loops!

✗ Traditional DB

Exact keyword match only. Query "loops" finds only docs containing "loops". No semantic understanding of concepts.

✓ ChromaDB Vectors

Semantic similarity search. "Loops" finds iteration, for/while, repetition — ranked by meaning proximity, not just text.

📦 ChromaDB — PersistentClient

/chroma_db · Data survives server restarts · Per-user isolation

FieldValue
Collectionmemory_{userId} — unique per browser session
DocumentsConcatenated prompt + " " + response strings
EmbeddingsAuto-generated by ChromaDB default embedding function
IDsUUID v4 — unique per stored interaction
RetrievalTop 3 cosine-similar results per query
Metadata{ user_id: "user_..." } for isolation

Session ID Flow — Generated once on first visit as user_ + timestamp. Stored in localStorage. Sent as userId on every request. Ensures an isolated ChromaDB collection per browser session, so no user ever bleeds context into another.

A.I. Perspective

This system showcases crucial AI concepts like large search spaces, semantic pruning, combinatorial efficiency, and heuristic retrieval. RAG is not just an API call — it is a pipeline that grows smarter with use. Vector embeddings allow the system to reason about meaning, not keywords, making follow-up conversations feel genuinely intelligent. It's a foundational pattern for any production AI assistant.
